First Step : Installing Tomcat 8.0 in eclipse.

  • Download tomcat 8.0 server from https://tomcat.apache.org/download-80.cgi
  • Extract it and put it in a local drive
  • Open Eclipse-Neon and go to Windows-->Preferences-->Server-->Runtime Environment . Then Click "Add" and select the Tomcat 8.0 server path
  • Click "OK"

Second Step : Create Maven Web-app Project.

  • In Eclipse go to File-->New-->Maven Project
  • Select webapp project from archtype

    • if it comes up with the following error: 1) error 2) no java folder inside the src folder

    Then do the following:

    Select Project and Go to Properties and check Tomcat 8.0

    Create new "java" folder inside src folder

Fourth Step : Understanding the WORKFLOW.

First Web.xml file will be loaded by Tomcat and then CXF will read the following configuration file :

As per configuration CXF will load the transport Servlet and also CXF will server its service as per following mapping configuration

Which means CXF will serve if the url pattern starts with /services. If you do not put /services then it will act as regular dynamic web application. It will not as restfull webservice. Thats why we see when http://localhost:8080 has entered index.jsp starts loads up. If you want to work with restfull service then you have to put http:localhost:8080/{project-name}/services .

After that CXF will be looking for bean.xml file inside WEB-INF folder. And the following bean and ref configuraton it will read the class and do according the annotation you provided in the class.

For bean.xml and web.xml go to the project and take a look at that sample
